# 6 Hjälpa och irritera

## Städa

Städa bland dina tidigare uppgifter. Ta bort kod som inte används. Lägg alla program i en mapp.

## Lägg upp dina projekt på github

- Skapa ett konto på github.com
- Skapa ett repo på github som du döper till **AcceleratedLearning**
- Clone'a repot till en mapp på din dator: **C:\Project\AcceleratedLearning**
- Flytta alla dina program till denna mapp
- Lägg till en ignore-fil.
- Stage. Commit.
- Skicka upp projektet till github mha push.

## Hämta hem kollegas projekt

- Hämta hem en kollegas projekt mha Clone.
- Testkör programmet.
- Försök lista ut hur det funkar. 


## Lägg till kommentarer

- Jobba vidare med kollegans projekt.
- Skriv kommentarer i koden. Gör en commit med meddelandet “Added comments”


## Ge feedback

Jobba vidare med kollegans projekt.

Ge en eller flera förslag på förbättringar. Gör det genom att skriva kommentarer med todo, t.ex:

// TODO: Använd ett tydligare variabelnamn

Commit’a med meddelandet “My suggestions” 

## Buggar och förbättringar

Jobba vidare med kollegans projekt.

Nu ska du göra ett par förbättringar och lägga in en bugg. Välj själv vilken ordning du vill göra commit (så kollegan inte vet vilken av comittarna som är en bugg).

Skriv tydliga commit-meddelanden. När du smyger in buggen så skriv något lurigt commit-meddelande så kollegan inte anar oråd.

## Rätta

Försök lista ut vilken/vilka buggar som din kollega har ställt till med. Rätta till problemet.
